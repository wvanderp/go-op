# Art and UI Direction

## Visual Contrast

The game should pair hyper-realistic bio-industrial units and structures with a clinical, highly legible UI. The battlefield should feel tactile, wet, and alive. The interface should feel cold, measured, and analytical, as if the player is viewing the war through the perception stack of the network itself.

## Terrain Direction

Terrain is currently static. The world should look like normal countryside rather than a ruined wasteland. Fields, forests, ramps, and height differences matter because they support line of sight, firing angles, and liquid flow, not because the terrain itself is constantly deforming. Nature should remain mostly intact and readable even across repeated battles.

## Goop Presentation

Goop should look like a dark oily red industrial fluid. It should feel heavy, glossy, and semi-transparent, and it should remain readable when moving through channels or pooling into puddles. Power goop should read as the same substance in a denser, more volatile state rather than as a different material entirely.

## Unit and Structure Style

Units should look like partially domesticated goop-creatures stabilized by technology and metal. They should be readable and appealing rather than grotesque. Structures should feel grown and engineered at the same time, as if biotech has been standardized into infrastructure. The baseline silhouette language should communicate bred utility animals first and military assets second.

## Interface Direction

The UI should take clear inspiration from Supreme Commander. Players should be able to open multiple live windows onto the same battle, including on separate screens. Every window should be functionally equal, meaning any window can become any mode that another window can become.

## Workstation Model

The window system should be organized around battlefield jobs rather than around fixed camera types. Any window can become a tactical control station, logistics authoring surface, intelligence view, production screen, or strategic overview. That keeps the multi-window system coherent while still supporting shared control and specialized roles.

## Shared Control Readability

Because multiplayer is pure shared command with no ownership, the UI needs strong pings, planning tools, and visible intent markers. The interface should make cooperation legible without forcing hard locks on units or structures.

## Intended Uses of Extra Windows

These windows are player-defined tools. A player might keep one on the base, one on the frontline, one on a zone map, one as an oversized minimap, or one on a specific objective. The point is to support a command-desk fantasy rather than a single-camera battlefield view.
